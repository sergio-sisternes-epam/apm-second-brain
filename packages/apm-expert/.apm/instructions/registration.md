# apm-expert Registration Metadata

This document describes the registration information a consumer must supply
when registering the apm-expert agent with the Agent Knowledge Network after
installing this package.

## Required registration values

```yaml
agentId: to-be-generated          # generated by the consumer's AKN instance
projectId: to-be-configured-by-consumer  # the consumer's project identifier
capability: apm-expert.answer.v1
transport: copilot-app
description: >
  APM documentation specialist answering questions about the Agent Package
  Manager using a vendored OKF knowledge corpus.
```

## Registration command

After installing this package, run:

```bash
apm install sergio-sisternes-epam/apm-second-brain/packages/agent-knowledge-network
# then use the akn-register skill to register:
# invoke akn-register with the values above
```

## Notes

- **Registration is explicit and idempotent.** Calling `akn-register` with the
  same `agentId` and `capability` more than once is safe.
- **agentId** must be generated by your AKN instance; the placeholder above is
  intentional and must be replaced before registration.
- **projectId** must match the project where the consumer deploys the agent.
- The `capability` identifier (`apm-expert.answer.v1`) is stable and part of
  the public API contract for this package.
- **Transport** is `copilot-app` for the reference deployment. Adjust if
  deploying via a different transport.
