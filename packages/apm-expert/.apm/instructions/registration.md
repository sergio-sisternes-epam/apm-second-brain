# apm-expert Registration Metadata

> **Pre-registration checklist -- registration MUST NOT proceed until ALL
> gates below have passed:**
>
> - [ ] Corpus built (run `apm run build-corpus` or follow `docs/knowledge-build.md`)
> - [ ] OKF conformance validated (`apm run validate-corpus` exits 0)
> - [ ] Integrity hash verified (MANIFEST.json `integrityHash` matches concepts/)
> - [ ] All conformance tests pass (`pytest packages/apm-expert/tests/ -q`)
> - [ ] `active` pointer is NOT `none` (corpus is live)
>
> Registering before these gates pass results in an agent that returns
> `quality: unanswered` for every query.

This document describes the registration information a consumer must supply
when registering the apm-expert agent with the Agent Knowledge Network after
installing this package.

A complete synthetic example of the registration record is provided at
`tests/fixtures/sample-registration.json`.

## Required fields

| Field | Required | Description |
|-------|----------|-------------|
| `agentId` | generated | UUID v4 generated by akn-register on first registration; stable thereafter |
| `displayName` | yes | Human-readable agent name, e.g. `"APM Expert"` |
| `owner` | yes | Owner identifier, e.g. team name |
| `project` | yes | Project short name |
| `transport.type` | yes | `copilot-app` for the reference deployment |
| `transport.projectId` | yes | Copilot App project ID -- **deduplication key used by akn-register** |
| `capabilities` | yes | List of capability objects (see schema below) |
| `knowledgeRoots` | no | List of knowledge root objects (optional) |
| `constraints` | no | List of constraint strings; defaults to `[]` |

## Capability object schema

```json
{
  "id": "apm-expert.answer.v1",
  "version": "0.1.0",
  "interactionMode": "conversational"
}
```

## Revision semantics

`akn-registry-io` uses optimistic concurrency. On every `upsert` call:

1. `akn-register` reads the registry with `op: read` to get the current `revision`.
2. `akn-register` calls `akn-registry-io` with `op: upsert` and
   `expectedRevision: <current stored revision>`.
3. `akn-registry-io` checks that `stored revision == expectedRevision`.
   If they differ (concurrent write), the operation is rejected.
4. On success, `akn-registry-io` increments `revision` by 1.

**Important:** `expectedRevision` must equal the current stored revision --
not `stored_revision + 1`. The increment is performed by `akn-registry-io` on success.

## Deduplication

`akn-register` deduplicates by `transport.projectId`. Calling `akn-register`
again with the same `projectId` updates the existing record in place (outcome:
`updated`) rather than creating a duplicate. The `agentId` is preserved.

## Registration command

After installing this package, register using the `akn-register` skill:

```
invoke akn-register with:
  displayName: "APM Expert"
  owner:       <your team>
  project:     <your project>
  transport.projectId: <your Copilot App project ID>
  capabilities:
    - id: apm-expert.answer.v1
      version: "0.1.0"
      interactionMode: conversational
```

Registration is **idempotent**: invoking `akn-register` with the same
`transport.projectId` more than once updates the record without creating
duplicates.
