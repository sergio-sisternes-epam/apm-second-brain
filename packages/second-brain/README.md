# second-brain

> **Status: Experimental / Under Development**

Convenience meta-package that installs the second-brain provider bundle
(learn + think sides) in a single `apm install` command.

Part of the [apm-second-brain](../../README.md) public demo monorepo.

## Purpose

Installing this package pulls in:

- **second-brain-learn** -- provider-side write path (sb-learn-handler,
  sb-forget-handler, sb-learn-validate, sb-forget-validate)
- **second-brain-think** -- provider-side read path (sb-think-handler,
  sb-think-validate)

together with their transitive dependencies (second-brain-interfaces,
karpathy-wiki, open-knowledge-format).

## No primitives

This package contains **no skills, instructions, or other primitives of
its own**. There is no `.apm/` directory. The source of truth for this
meta-package is `apm.yml` and `apm.lock.yaml` only.

The dependency packages (second-brain-learn, second-brain-think, etc.) each
have their own `.apm/` directory which is the source of truth for their
primitives. When a consumer installs this package, `apm install` materialises
those primitives into generated target directories (`.agents/`, `.claude/`,
`.github/`) in the consumer project. These generated directories must not be
edited or committed -- they are regenerated by `apm install`.

## Installation

```bash
apm install sergio-sisternes-epam/apm-second-brain/packages/second-brain
```

This is equivalent to installing `second-brain-learn` and
`second-brain-think` individually and pulls in all transitive dependencies.
It installs the provider bundle only; see the required configuration steps below.

## What installing this package does NOT do

Installing `second-brain` alone is not sufficient to run a provider.
You must also complete the following steps:

1. **Initialise a wiki** -- run `kw-wiki-init` from `karpathy-wiki` to create
   the wiki bundle at your chosen path.
2. **Configure the provider** -- your orchestrator must supply the `wiki_root`
   path to the internal handler skills at invocation time. Users do not pass
   `wiki_root` directly; it is provider/adapter configuration.
3. **(Optional) Register for discovery** -- if you want your provider to be
   discoverable via `agent-knowledge-network`, install it separately and run
   `akn-register`. `agent-knowledge-network` is intentionally not a dependency
   of `second-brain`; install it if and only if you need local discovery.

   ```bash
   apm install sergio-sisternes-epam/apm-second-brain/packages/agent-knowledge-network
   ```

## User-facing skills

After installing and configuring, consumers call the **public client skills**
from `second-brain-interfaces` (not the internal handler skills directly):

| Skill | Capability | Description |
|-------|-----------|-------------|
| `brain-think` | `second-brain.think.v1` | Ask a question; citation-backed answer |
| `brain-learn` | `second-brain.learn.v1` | Ingest new knowledge |
| `brain-forget` | `second-brain.forget.v1` | Tombstone/archive a concept (v1: archive only; no destructive deletion) |

The internal handler skills (`sb-learn-handler`, `sb-think-handler`, etc.)
are invoked by the framework -- direct user invocation is disabled on those.

## Dependencies

| Package | Role |
|---------|------|
| `second-brain-learn` | Provider write path |
| `second-brain-think` | Provider read path |
